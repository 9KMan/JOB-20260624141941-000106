# AI Report Writer (PySide6 Desktop App)

**Built by: KMan | AI-Augmented Engineering Factory**

## Business Problem Solved

School psychologists spend hours writing assessment reports from raw student documents (evaluations, observations, parent/teacher input). Existing AI report-writers produce low-quality output that requires heavy editing — defeating the time-savings purpose.

This desktop app lets a psychologist:
1. **Upload PDFs** of student records, evaluations, observations
2. **Click through** a 4-step workflow (no technical skill required)
3. **Get a high-quality narrative draft** generated by AWS Bedrock Claude, using prompts the psychologist has personally written
4. **Edit the narrative** in-app with familiar text controls
5. **Export a finished Word document** matching the practice's template

The key insight: the psychologist owns the prompts and templates (in plain `.txt` and `.json` files). The developer just wires up the plumbing. This means the psychologist can improve output quality over time without writing code.

## Architecture

```
+---------------------+        +------------------+
|  PySide6 GUI        |  <-->  |  Core Pipeline   |
|  (4 pages)          |        |                  |
+---------------------+        |  - config loader |
        |                       |  - prompt loader |
        v                       |  - pdf_extract   |
+---------------------+        |  - bedrock       |
|  config/app.json    |        |  - docx_export   |
|  config/prompts/*   |        +------------------+
+---------------------+                |
        |                              v
        |                       +------------------+
        +---------------------> |  AWS Bedrock     |
                                |  (Claude 3)      |
                                +------------------+
```

### Pipeline Flow

1. **Upload** → user selects PDF files via GUI
2. **Decisions** → user enters clinical decisions (interview type, focus)
3. **Extract** → `pdf_extract.extract_text()` parses each PDF
4. **LLM Call** → `bedrock.invoke()` sends extracted text + prompt template to AWS Bedrock Claude 3
5. **Review** → narrative displayed in editable text widget
6. **Export** → `docx_export.export_report()` writes Word document using practice template

## Tech Stack

| Layer | Tool |
|-------|------|
| GUI | PySide6 (Qt for Python) |
| LLM | AWS Bedrock (boto3) |
| PDF | pdfplumber, pypdf |
| Word | python-docx |
| Config | JSON + plain-text prompts |

## How to Update Behavior (Non-Code)

The psychologist can change app behavior without code changes:

- **Edit UI labels / page order:** `config/app.json`
- **Update LLM prompts:** `config/prompts/*.txt`
- **Switch model:** edit `aws.model_id` in `config/app.json`

## Running Locally
